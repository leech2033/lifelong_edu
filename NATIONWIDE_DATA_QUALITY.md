# 전국 기관 데이터 품질 점검

## 현재 상태

- 전국 기관 목록은 `client/src/data/institutionsNationwide.ts`를 단일 소스로 사용한다.
- 서비스 레이어에서 `regionId`, 정규화된 `region`, 기본 `status`, `sourceQuality`를 부여한다.
- 원천 JSON과 인라인 데이터 중 일부 문자열은 여전히 인코딩이 깨져 있다.

## 이번 보완

- 지역 필터 기준을 문자열 비교에서 `regionId` 비교로 변경했다.
- 시도명 표시는 `client/src/data/institutionRegions.ts`의 정규화된 한글명을 사용한다.
- 기관 데이터에는 다음 보정이 적용된다.
  - `regionId`: 기관 ID 범위로 추론
  - `region`: 정규화된 시도명으로 교체
  - `detail_region`: 깨진 값이면 시도명으로 대체
  - `status`: 깨진 값이면 `운영중`으로 대체
  - `sourceQuality`: 원천 문자열에 깨짐이 있으면 `needs_review`

## 남아 있는 이슈

- 기관명, 주소, 세부 지역명, 카테고리의 일부 값은 원천 데이터 재수집 없이는 완전 복구가 어렵다.
- 서울, 대전, 부산, 대구, 인천 JSON도 문자열 깨짐이 포함되어 있어 “JSON이라서 곧바로 검증 완료” 상태는 아니다.
- 아직 미수집 상태인 시도는 울산, 세종, 충북, 충남, 전북, 전남이다.

## 권장 후속 작업

1. 시도별 원천 파일을 UTF-8 기준으로 다시 수집하거나 변환한다.
2. 수집 스크립트 단계에서 문자셋 검증을 추가한다.
3. `sourceQuality = needs_review` 항목만 따로 추출하는 검수 화면 또는 CSV를 만든다.
4. 기관 상세 설명과 프로그램 문구는 원천 필드가 안정화된 뒤 지역별로 보강한다.
