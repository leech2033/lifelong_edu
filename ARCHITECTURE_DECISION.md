# 초기 아키텍처 결정

## 결정

현재 프로젝트는 `정적 데이터 기반 포털 우선` 전략으로 정리합니다.

## 이유

- 실제 백엔드 API가 아직 구현되어 있지 않습니다.
- 사용자 가치의 핵심은 "탐색 가능한 포털"이며, 운영 기능은 아직 프로토타입 단계입니다.
- 데이터 품질과 구조가 먼저 정리되지 않으면 API를 올려도 안정적으로 확장하기 어렵습니다.

## 현재 단계의 원칙

1. 프론트엔드는 정적 데이터와 공통 타입을 기준으로 안정화합니다.
2. 데이터 구조를 먼저 표준화한 뒤, 이후 API 응답 형태로 자연스럽게 이행할 수 있게 합니다.
3. 관리자 대시보드는 MVP 범위 밖의 실험적 UI로 취급합니다.
