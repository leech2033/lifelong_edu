# LifelongEduPortal MVP 기능정의서

## 1. MVP 목표

MVP의 목표는 다음과 같습니다.

`사용자가 시니어 친화적인 웹 서비스에서 평생교육기관, 온라인 강좌, 지역 프로그램을 한 곳에서 검색하고 탐색할 수 있도록 한다.`

현재 MVP는 운영 플랫폼 전체가 아니라, 우선적으로 정보 탐색 중심의 포털을 목표로 합니다.

## 2. MVP 제품 정의

### 범위에 포함

- 대국민 대상 웹 포털
- 기관 검색 및 상세 조회
- 온라인 강좌 목록 및 상세 조회
- 지역 프로그램 목록 및 상세 조회
- 다모아형 강좌 목록 화면
- 성과자료 및 아카이브 화면

### MVP 범위에서 제외

- 기관 관리자 대시보드의 실제 운영 기능
- 실제 강좌 신청 및 결제 흐름
- 출석 관리
- 보고서 제출 업무 흐름
- 권한 기반 관리자 기능
- 고도화된 개인화 기능

## 3. 대상 사용자

### 주요 사용자

- 시니어
- 교육 또는 복지 프로그램을 찾는 지역 주민

### 보조 사용자

- 기관 또는 성과정보를 확인하는 공공 실무자
- 포털 노출을 원하는 참여 기관

## 4. MVP 성공 기준

다음 조건을 만족하면 MVP가 성공한 것으로 봅니다.

1. 사용자가 지역 또는 키워드로 원하는 기관을 찾을 수 있다.
2. 사용자가 여러 외부 사이트를 전전하지 않고 온라인 강좌나 지역 프로그램을 찾을 수 있다.
3. 사용자가 대상, 기간, 위치, 비용, 신청 방법 등 의사결정에 필요한 기본 정보를 확인할 수 있다.
4. 사용자가 연간 성과집과 우수사례 자료를 한 곳에서 열람할 수 있다.
5. 서비스가 데스크톱과 모바일에서 모두 무리 없이 동작한다.

## 5. MVP 핵심 사용자 흐름

### 흐름 A. 기관 탐색

1. 사용자가 기관 페이지에 진입한다.
2. 지역 또는 키워드로 기관을 필터링한다.
3. 목록형, 카드형, 지도형 화면으로 탐색한다.
4. 기관 상세 페이지를 연다.
5. 주소, 전화번호, 태그, 관련 프로그램, 홈페이지 등을 확인한다.

### 흐름 B. 온라인 강좌 탐색

1. 사용자가 온라인 학습 페이지에 진입한다.
2. 강좌 목록을 살펴본다.
3. 상세 페이지를 연다.
4. 주제, 요약, 썸네일, 외부 시청 링크 또는 임베드 정보를 확인한다.

### 흐름 C. 지역 프로그램 탐색

1. 사용자가 지역 프로그램 페이지에 진입한다.
2. 지역, 운영 방식, 대상 등으로 필터링한다.
3. 프로그램 상세를 연다.
4. 운영기관, 일정, 대상, 신청 방법을 확인한다.

### 흐름 D. 자료 아카이브 이용

1. 사용자가 성과자료 또는 리소스 페이지에 진입한다.
2. 연도별 성과집이나 우수사례 자료를 선택한다.
3. 파일을 열거나 다운로드한다.

## 6. MVP 화면 목록

### 필수 화면

- 홈
- 기관 목록
- 기관 상세
- 온라인 강좌 목록
- 온라인 강좌 상세
- 지역 프로그램 목록
- 지역 프로그램 상세
- 다모아형 강좌 목록
- 성과집 목록
- 우수사례 및 리소스 페이지
- 404 페이지

### 있으면 좋은 화면

- 안내형 신청/접수 페이지

### MVP에서 필수 아님

- 실제 운영 기능이 연결된 대시보드 하위 화면

## 7. 기능 요구사항

## FR-1. 홈

- 서비스 목적이 명확하게 전달되어야 한다.
- 온라인 학습과 지역 프로그램으로 바로 진입할 수 있어야 한다.
- 최신 또는 추천 콘텐츠 미리보기를 제공해야 한다.
- 시니어 친화적인 문구와 읽기 쉬운 시각 구조를 가져야 한다.

## FR-2. 기관 목록

- 지역 선택 기능을 제공해야 한다.
- 기관명 또는 주소 기준 키워드 검색을 제공해야 한다.
- 각 기관 항목에 최소 다음 정보를 표시해야 한다.
  - 기관명
  - 지역/세부 지역
  - 주소
  - 전화번호
  - 태그 또는 분류
  - 상태 정보가 있으면 함께 표기
- 최소 목록형과 카드형 보기를 지원해야 한다.
- 지도형 보기는 MVP에서 시각적 플레이스홀더여도 무방하나, 사용자가 오해하지 않도록 표현해야 한다.

## FR-3. 기관 상세

- 기관명, 분류, 주소, 전화번호, 홈페이지, 소개, 운영 정보가 보여야 한다.
- 대표 이미지가 있으면 함께 제공해야 한다.
- 관련 프로그램 또는 대표 강좌 정보가 있으면 함께 보여야 한다.
- 사용자가 기관 목록으로 쉽게 돌아갈 수 있어야 한다.

## FR-4. 온라인 강좌 목록/상세

- 제목, 카테고리, 플랫폼, 썸네일, 짧은 설명이 보여야 한다.
- 각 항목의 상세 정보를 열 수 있어야 한다.
- 외부 시청 링크 또는 임베드 미디어를 지원해야 한다.
- 태그, 추천 대상 등 사용자가 이해하기 쉬운 메타데이터를 보여줘야 한다.

## FR-5. 지역 프로그램 목록/상세

- 지역, 운영기관, 프로그램명, 대상, 일정, 운영 방식, 신청 방법을 보여야 한다.
- 여러 프로그램을 탐색할 수 있어야 한다.
- 개별 프로그램 상세 보기 기능이 있어야 한다.

## FR-6. 다모아형 강좌 목록

- 게시판형 또는 카드형 구조로 강좌 목록을 제공해야 한다.
- 상태, 운영기관, 카테고리, 일정, 대상, 수강료, 접수 기간을 보여야 한다.
- 접수중, 마감, 예정 상태를 시각적으로 구분해야 한다.
- MVP에서는 신청 버튼이 실제 결제/신청 흐름이 아니라 안내용이거나 외부 링크여도 된다.

## FR-7. 성과자료 및 리소스

- 연도별 성과집 목록을 제공해야 한다.
- 실제 파일이 있는 경우 열람 또는 다운로드가 가능해야 한다.
- 우수사례, 미디어, 리소스 페이지가 일관된 내비게이션 안에 있어야 한다.

## FR-8. 내비게이션 및 레이아웃

- 공통 상단 내비게이션이 있어야 한다.
- 모바일과 데스크톱 모두 반응형으로 동작해야 한다.
- 뒤로가기 동선과 섹션 제목이 명확해야 한다.

## FR-9. 데이터 관리

- 기관, 강좌, 프로그램 데이터의 필드 구조를 가능한 한 일관되게 맞춰야 한다.
- 현재 로컬 JSON/TS 목업 데이터를 향후 API 응답으로 교체할 수 있어야 한다.
- 원천 데이터와 서비스 노출 데이터를 논리적으로 분리해야 한다.

## 8. 비기능 요구사항

### NFR-1. 접근성

- 큰 제목과 읽기 쉬운 텍스트 구조
- 충분한 대비를 가진 버튼과 라벨
- 누르기 쉬운 인터랙션 영역
- 쉬운 내비게이션 문구
- 과도하게 복잡하지 않은 화면 구성

### NFR-2. 성능

- 정적 데이터 또는 사전 번들된 데이터 기준으로 초기 로딩이 빨라야 한다.
- 이미지와 첨부 파일 로딩이 핵심 탐색 흐름을 방해하지 않아야 한다.

### NFR-3. 유지보수성

- 데이터 구조가 표준화되어야 한다.
- 목업 데이터는 페이지 내부 하드코딩에서 점진적으로 분리되어야 한다.
- 라우팅과 기능 책임 범위가 명확해야 한다.

### NFR-4. 콘텐츠 품질

- 릴리스 전 한글 인코딩 문제를 해결해야 한다.
- 화면 전반의 명칭과 표현을 통일해야 한다.

## 10. 우선순위

- 홈

- 온라인 강좌 목록/상세

- 반응형 공통 내비게이션

### P1. 강력 권장

- 다모아형 강좌 목록
- 검색/필터 고도화
- 대표 이미지 및 다운로드 자료 정리

### P2. MVP 이후

- 실제 관리자 대시보드
- 로그인/인증
- DB 기반 CRUD
- 실제 신청 흐름
- 출석/보고 기능
- 알림/즐겨찾기/최근 본 항목

## 11. 수용 기준

### 기관 기능 수용 기준

- 사용자가 `/institutions` 페이지를 열 수 있어야 한다.
- 사용자가 최소 1개 이상의 지역 필터를 사용할 수 있어야 한다.
- 사용자가 키워드 검색을 수행할 수 있어야 한다.
- 사용자가 목록 또는 카드에서 상세 페이지로 이동할 수 있어야 한다.

### 온라인 기능 수용 기준

- 사용자가 여러 개의 온라인 강좌를 탐색할 수 있어야 한다.
- 사용자가 최소 1개 이상의 강좌 상세 페이지를 열 수 있어야 한다.
- 상세 페이지에 시청 여부를 판단할 수 있는 정보가 있어야 한다.

### 지역 프로그램 기능 수용 기준

- 사용자가 여러 개의 지역 프로그램을 탐색할 수 있어야 한다.
- 사용자가 대상, 일정, 신청 방법을 식별할 수 있어야 한다.

### 자료 기능 수용 기준

- 사용자가 최소 1개 이상의 성과집 PDF를 정상적으로 열거나 다운로드할 수 있어야 한다.

### 품질 수용 기준

- 눈에 띄는 한글 깨짐 문제가 제거되어야 한다.
- 주요 내비게이션 라벨이 한국어 기준으로 이해 가능해야 한다.
- 모바일 폭과 데스크톱 폭에서 모두 레이아웃이 무너지지 않아야 한다.

## 12. MVP 리스크

- 현재 백엔드는 실제 데이터 API를 거의 제공하지 않는다.
- 일부 상세 페이지는 여전히 플레이스홀더 데이터에 의존한다.
- 지역별 원천 데이터의 품질과 구조가 다르다.
- 저장소에 운영용 자산과 원천 파일이 혼재되어 있다.
- 대시보드 UI가 실제 구현 범위를 과대 인식하게 만들 수 있다.

## 13. 권장 개발 순서

1. 인코딩과 화면 문구를 먼저 정리한다.
2. 기관, 강좌, 프로그램의 공통 데이터 구조를 표준화한다.
3. 대국민 화면의 라우팅과 콘텐츠를 안정화한다.
4. 기관 상세 및 강좌 상세 품질을 정리한다.
5. 성과자료/리소스 페이지를 마무리한다.
6. 다모아 영역을 안내형으로 둘지, 실제 신청형으로 확장할지 결정한다.
7. MVP 이후 DB/API와 관리자 기능을 도입한다.

## 14. 한 줄 정의

`이 MVP는 시니어 친화적인 평생교육 정보 탐색 포털로서, 사용자가 기관, 강좌, 지역 프로그램, 성과자료를 한 곳에서 찾을 수 있도록 하는 것을 목표로 한다.`
