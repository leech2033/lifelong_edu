# 리팩토링 사전 점검 기록

## 1. 현재 기준선

- 작업 경로: `D:\05. git\lifelong_edu`
- 타입체크: `npm.cmd run check` 통과
- 서버 상태: Express 진입점은 있으나 실제 API 라우트는 비어 있음
- 데이터 상태: 정적 JSON/TS 데이터와 페이지 내부 목업 데이터가 혼재
- 자산 상태: `attached_assets`, `client/public`, 루트 CSV/JSON이 혼재

## 2. 현재 확인된 주요 리스크

### 구조

- 페이지 내부에 데이터가 직접 선언된 경우가 많음
- 공통 타입이 부족해 데이터 구조가 파일마다 암묵적으로 관리됨
- 포털 화면과 관리자 대시보드가 같은 제품 범위 안에서 섞여 있음

### 콘텐츠

- 일부 파일에 한글 깨짐 또는 미정리 문구가 남아 있음
- 영어/한글 라벨이 혼재되어 있음

### 아키텍처

- 현재는 정적 포털에 가깝고, API/DB는 사실상 미구현 상태
- 리팩토링 전에 "정적 포털 우선" 여부를 확정해야 함

## 3. 리팩토링 진행 원칙

1. 먼저 사용자에게 보이는 핵심 화면의 문구와 데이터 참조를 안정화한다.
2. 공통 타입과 데이터 접근 함수를 도입해 페이지 내부 결합도를 낮춘다.
3. 큰 페이지를 컴포넌트 단위로 쪼개기 전에 데이터 책임을 먼저 분리한다.
4. 관리자 기능은 MVP 포털 정리와 분리해서 다룬다.

## 4. 이번 정리 범위

- 기준선 문서화
- 핵심 공통 화면 문구 정리
- 온라인/지역 프로그램 데이터 타입 추가
- 데이터 접근 헬퍼 도입
- 일부 상세 페이지 책임 분리
- 초기 아키텍처 결정 문서화
